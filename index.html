Step 01
1
APIキーを絶対に漏らさない方法 — .envファイル管理

まずAPIキーをコードに直書きしているかどうかを確認してください。sk-ant-xxxx という文字列がコードの中に直接書いてあったら、それは今すぐ変えてください。これが一番よくある事故の原因です。

Terminal — プロジェクトフォルダ
# ① まず .env ファイルを作る
touch .env
 
# ② APIキーを .env に書く（コードには書かない！）
echo 'ANTHROPIC_API_KEY=sk-ant-xxxxx' >> .env
 
# ③ .gitignore に .env を追加して Git に上げない
echo '.env' >> .gitignore
 
✓ これでAPIキーがGitHubに上がることはありません
▲ ターミナルでの設定手順。3行のコマンドで完了します。
.gitignore に追加することで、GitHub へのうっかり公開を防止。
.gitignore — Gitで無視するファイル一覧
# .gitignore に必ず追加すること
.env
.env.local
.env.production
 
# node_modules も必ず入れておく
node_modules/
__pycache__/
*.pyc
▲ .gitignore の記述例。.env 系のファイル名を全部書いておくのが安全。
「.env.local」「.env.production」も忘れずに追加しましょう。
❌ NG：api_key = "sk-ant-xxxx" をコードに直書き
✅ OK：api_key = os.getenv("ANTHROPIC_API_KEY") で環境変数から読む
Step 02
2
Claude Codeの権限設定を最小限に — CLAUDE.md活用

Claude Codeはデフォルトでかなり広い範囲を操作できます。プロジェクトフォルダに CLAUDE.md を置くことで、AIに対して「ここだけ触っていい」と明示できるんですよ。

CLAUDE.md — プロジェクト直下に置くAIへの指示書
# CLAUDE.md — Claude Codeへの指示書
 
## 許可する操作
- src/ フォルダ内のファイル編集
- tests/ フォルダ内のテスト実行
- package.json の依存関係更新
 
## 禁止する操作
- .env ファイルの読み取り・書き込み
- rm -rf などの削除コマンド実行
- 本番環境（production）へのデプロイ
- GitHub への直接プッシュ
 
## 作業ルール
- 変更前に必ず確認を取ること
- 大きな変更はブランチを切って作業
▲ CLAUDE.md の記述例。プロジェクトルートに置くだけで機能します。
「禁止する操作」の欄が特に重要。削除・デプロイは必ず明示的に禁止しましょう。
💡 CLAUDE.mdはAIへの「契約書」です。書いておくだけで、Claude Codeが勝手な操作をする確率が大幅に下がります。
Step 03
3
ファイル削除事故を防ぐ — 危険コマンドを使わない

これ、初心者が一番やりがちなミスです。「権限エラーが出た、じゃあ権限を全部スキップしよう」って考えてしまう。これは鍵のかかった家に入りたくて、壁を壊してるようなものです。

Terminal — 危険な使い方（絶対NG）
# ❌ これは絶対にやらない
claude --dangerously-skip-permissions
 
# ❌ sudoでClaudeを動かすのも危険
sudo claude
 
# なぜ危ないか？
# → AIが確認なしに何でも実行できてしまう
# → rm -rf / も実行できてしまう（最悪の場合）
▲ 絶対にやってはいけないコマンドのパターン。
権限スキップはどんな理由があっても使わないのが正解です。
Terminal — 安全な使い方
# ✅ 通常の起動（これでいい）
claude
 
# ✅ 特定フォルダだけで動かす
cd ./my-project && claude
 
# ✅ 権限エラーが出たら原因を調べる
ls -la ~/.claude/
cat ~/.claude/settings.json
 
✓ 権限は「最小限」が鉄則
▲ 安全な起動方法。まず通常の「claude」コマンドで起動して、
権限エラーが出た場合は原因を調べてから対処しましょう。
Step 04
4
Gitで「バックアップ」を常に確保する

Claude Codeがコードを変更する前に、Gitでコミットしておく。これだけで「事故っても元に戻せる」状態になります。当然ですよね。ファイルを壊されても怖くない状態を先に作る。これがプロの作法です。

Terminal — Claude Code作業前の安全確保フロー
# ① 現在の状態をすべてコミット（作業前の儀式）
git add -A && git commit -m "before claude: 作業前の保存"
 
# ② 新しいブランチを切ってからClaudeを起動
git checkout -b feature/claude-work
claude
 
# ③ 問題があったらいつでも元に戻せる
git checkout main
git branch -D feature/claude-work
 
✓ これで事故ゼロ。怖いものなしです。
▲ Claude Code作業前の安全確保フロー。コミット → ブランチ作成 → 起動の順番。
失敗してもブランチを捨てるだけで元通りになります。
Step 05
5
settings.jsonで危険なコマンドをブロックする

Claude Codeには設定ファイルがあります。ここで「絶対に実行してはいけないコマンド」をブラックリストに入れておくことができるんですよ。有料にしておきたいところですが、そのまま使えるテンプレートをお見せします。

~/.claude/settings.json — コピペOKテンプレート
{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)",
      "Bash(python:*)",
      "Read(**)",
      "Write(src/**)"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(sudo:*)",
      "Bash(curl * | bash:*)",
      "Bash(wget * | bash:*)"
    ]
  }
}
▲ settings.json のセキュリティ設定テンプレート。allowで許可、denyで禁止。
「rm -rf」「sudo」「curl | bash」の3つは必ず禁止リストに入れてください。
✅ このテンプレートをコピペするだけで、危険な操作を自動でブロックできます。初心者のうちは「deny」の項目を増やす方向で管理するのが安全です。
